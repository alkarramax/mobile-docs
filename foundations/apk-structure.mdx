# Estrutura e Organização de um APK

Essa página descreve a estrutura de um APK (**A**ndroid **P**ackage **K**it), que é o formato de arquivo usado para distribuir e instalar aplicativos no Android. Apesar do sufixo, o APK é um tipo de JAR (**J**ava **Ar**chive) que contém todos os recursos e arquivos necessários para instalar e executar um aplicativo no Android.

Sabendo que JAR é um tipo de ZIP, é possível modificar o sufixo de um arquivo APK para `.zip`, descompactá-lo e ver a estrutura de diretórios e arquivos que compõem o aplicativo.

## Diretórios raiz

Diretórios essenciais que compõem um APK

|            |                                                                                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `META-INF` | Essa pasta contém informações de verificação e é gerada durante o processo de assinatura do aplicativo. O que significa que qualquer modificação no APK (mesmo a simples substituição de um ícone) exige que o APK seja re-assinado.                                                                       |
| `lib`      | Bibliotecas nativas compiladas para diferentes arquiteturas de dispositivos.                                                                           |
| `res`      | Recursos predefinidos da aplicação, como arquivos XML que definem lista de cores, layout da interface de usuário, fontes, valores, etc.                |
| `assets`   | Recursos estáticos (Imagens, vídeos, documentos e etc) que o aplicativo pode acessar em tempo de execução.                                             |
| `kotlin`   | Diretório presente apenas se o aplicativo foi desenvolvido em [Kotlin](https://developer.android.com/kotlin/). Ele contém dados específicos do Kotlin. |

## Arquivos raiz

Arquivos essenciais que são responsaveis por definir a estrutura do aplicativo, componentes, permissões, recursos e configurações gerais.

|                                                        |                                                                                                                               |
| ------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| `MANIFEST.MF`                                          | Lista de nomes/hashes (geralmente SHA-256 em Base64) para todos os arquivos do APK.                                            |
| `CERT.SF`                                              | Lista de nomes/hashes das linhas correspondentes no arquivo `MANIFEST.MF`.                                                    |
| `CERT.RSA`                                             | Chave pública e a assinatura do arquivo `CERT.SF`.                                                                            |
| [`AndroidManifest.xml`](/foundations/android-manifest) | Informações essenciais sobre o aplicativo: componentes, permissões, requisitos, nome do pacote e o entry point do aplicativo. |
| `classes.dex`                                          | Todas as classes Java no formato de arquivo dex (Dalvik Executable), para serem executadas pelo Android Runtime.              |
| `resources.arsc`                                       | Recursos pré-compilados. Ele guarda informações que ligam o código (`classes.dex`) aos recursos (`res`).                      |
